Mesure continue et enregistrement dans un fichier

L'exemple de projet Mesure continue et enregistrement dans un fichier acquiert des mesures en continu et les enregistre sur le disque. Il exécute cinq boucles en parallèle :

  • Gestion d'événements (Main.vi) — La boucle de gestion d'événements (BGE) qui produit des messages basés sur les événements qui ont lieu sur la face-avant, comme un clic sur Démarrer ou sur Paramètres.
  • Communication par messages de l'interface utilisateur (Main.vi) — Une boucle de gestion de messages (BGM) qui reçoit des messages de la BGE et y répond en envoyant des messages aux autres BGM.
  • Acquisition des données (Acquisition.lvlib:Acquisition Message Loop.vi) — Une BGM qui acquiert des données en continu. Par défaut, ce modèle simule les données acquises.
  • Enregistrement des données (Logging.lvlib:Logging Message Loop.vi) — Une BGM qui enregistre en continu les données acquises.
  • Affichage des données (Main.vi) — Une boucle While qui met le graphe déroulant à jour avec les données acquises.

Cet exemple de projet comporte aussi une boîte de dialogue Paramètres (Settings.lvlib) que vous pouvez utiliser pour configurer l'application.

Configuration système requise

Système de base, complet ou professionnel de LabVIEW. Cet exemple de projet est conçu pour être utilisé avec NI-DAQmx, un driver d'instruments ou d'autres drivers.

Organigramme du projet

Cas d'usage

L'exemple de projet Mesure continue et enregistrement dans un fichier est conçu pour une application de mesure en continu qui requiert une interface utilisateur réactive ; autrement dit, les utilisateurs doivent pouvoir cliquer sur les boutons même si l'application est en train d'exécuter une autre commande.

Exécution de cet exemple de projet

  1. Dans la fenêtre de l'Explorateur de projet, ouvrez et exécutez le VI Main.vi.
  2. Cliquez sur Démarrer. Le programme commence à acquérir des données waveform simulées.
  3. Cliquez sur les autres boutons de la face-avant pour explorer l'exemple de projet.

Modification de cet exemple de projet

Ajout de code d'acquisition de données

Vous devez modifier l'exemple de projet pour acquérir des données du matériel. Effectuez les étapes suivantes pour faire ces modifications :

  1. Ajoutez des refnums de matériel à la commande Acquisition.lvlib:Hardware Configuration.ctl. Vous pouvez, par exemple, utiliser les objets suivants :
    • Tâches DAQ
    • Voies DAQ
    • Sessions VISA
  2. Ajoutez le code d'initialisation du matériel au VI Acquisition.lvlib:Initialize Hardware References.vi. Vous pouvez, par exemple, utiliser ici les objets suivants :
    • Constantes nom de tâche DAQmx
    • VI DAQmx - Créer une voie virtuelle
    • (Driver d'instruments) VI Initialize
  3. Ajoutez le code de configuration du matériel au VI Acquisition.lvlib:Configure Hardware.vi. Vous pouvez, par exemple, utiliser les VIs suivants :
    • VI DAQmx - Cadencement
    • VI DAQmx - Déclenchement
    • (Driver d'instruments) VI Configure Measurement
    • (Driver d'instruments) VI Configure Autozero
  4. Ajoutez le code d'acquisition de données au VI Acquisition.lvlib:Acquire.vi. Vous pouvez, par exemple, utiliser ici les VIs suivants :
    • VI DAQmx - Lire
    • (Driver d'instruments) VI Read
  5. Ajoutez le code qui arrête l'acquisition de données au VI Acquisition.lvlib:Stop Acquisition.vi. Vous pouvez, par exemple, utiliser ici les VIs suivants :
    • VI DAQmx - Supprimer une tâche
    • (Driver d'instruments) VI Close

Personnalisation du code d'enregistrement des données

Si le comportement d'enregistrement par défaut ne convient pas aux besoins de votre application, vous pouvez modifier cet exemple de projet comme suit :

  • Pour spécifier où enregistrer les données, exécutez le VI Main.vi, cliquez sur Paramètres et utilisez la commande Chemin du fichier journal. Par défaut, ce modèle enregistre les données dans LabVIEW Data\Logged Data.tdms, LabVIEW Data étant le dossier LabVIEW Data.
  • Pour changer le mécanisme d'enregistrement des données, modifiez le VI Logging.lvlib:Logging Message Loop.vi. Vous pourriez, par exemple, modifier ce VI pour transférer en continu les données acquises via un réseau ou vers un disque.
  • Pour changer le code qui écrit les données sur disque, modifiez le VI Logging.lvlib:Log Data.vi. Vous pourriez, par exemple, utiliser les VIs Exporter des waveforms vers un fichier tableur ou Écrire dans un fichier tableur. Par défaut, ce modèle utilise les fonctions TDMS pour enregistrer les données dans un fichier .tdms.
  • Pour changer ou ajouter les refnums de chemin et de fichier nécessaires à l'enregistrement de données, modifiez la commande Logging.lvlib:Logging Configuration.ctl.
